üîß Prerequisites

  • A Google Cloud Project with billing enabled
  • gcloud CLI installed and initialized

  • A Dockerfile or a standard app structure (e.g., Node.js, Python)
  • IAM Roles: Cloud Build Editor, Cloud Run Admin, Service Account User

üõ†Ô∏è Step 3: Add Cloud Build Configuration

cloudbuild.yaml

steps:
  - name: 'gcr.io/cloud-builders/docker'
    args: ['build', '-t', 'gcr.io/$PROJECT_ID/my-app', '.']

  - name: 'gcr.io/cloud-builders/docker'
    args: ['push', 'gcr.io/$PROJECT_ID/my-app']

  - name: 'gcr.io/google.com/cloudsdktool/cloud-sdk'
    entrypoint: gcloud
    args:
      [
        'run', 'deploy', 'my-app',
        '--image', 'gcr.io/$PROJECT_ID/my-app',
        '--region', 'us-central1',
        '--platform', 'managed',
        '--allow-unauthenticated'
      ]

üåê Step 4: Push Your Code to Git

git init

üîê Step 5: Grant IAM Permissions to Cloud Build

PROJECT_ID=$(gcloud config get-value project)
PROJECT_NUMBER=$(gcloud projects describe $PROJECT_ID --format='value(projectNumber)')

gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member="serviceAccount:$PROJECT_NUMBER@cloudbuild.gserviceaccount.com" \
  --role="roles/run.admin"

gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member="serviceAccount:$PROJECT_NUMBER@cloudbuild.gserviceaccount.com" \
  --role="roles/iam.serviceAccountUser"

üîÑ Step 6: Create a Cloud Build Trigger

  1. Go to Cloud Build ‚Üí Triggers
  2. Click Create Trigger
  3. Source: Select GitHub or Cloud Source Repositories
  4. Repository: Connect and choose your repo
  5. Event: Push to a branch
  6. Branch Regex: ^main$
  7. Configuration: Use cloudbuild.yaml
  8. Click Create

üß™ Step 7: Test the CI/CD Pipeline

git push origin main

Go to Cloud Build ‚Üí History to view logs and confirm deployment.

üåç Step 8: Access Your Cloud Run App

gcloud run services describe my-app \
  --region us-central1 \
  --platform managed \
  --format='value(status.url)'

Open the returned URL in a browser to see your deployed app live!
